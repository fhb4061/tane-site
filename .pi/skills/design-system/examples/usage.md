# Design system usage

Use the skill when:
- introducing a new shared UI primitive
- refactoring repeated Tailwind utility clusters
- standardizing headings, spacing, or cards
- checking accessibility on shared components

Example prompts:
- "Is this component a shared primitive or page-only styling?"
- "What should the spacing scale be for sections and cards?"
- "Does this heading hierarchy match the document structure?"
- "Can we remove this repeated Tailwind cluster by extracting a primitive?"
